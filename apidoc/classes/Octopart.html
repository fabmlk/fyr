    fetchRaw

    (
    • url
    )
    Promise

    Defined in js/octopart.js:149

    Make an octopart request from a raw octopart url (or any url returning a json representation of the data: see demo version in resistor.view.js) and returns a promise representation of this request with: resolved: the data were correctly fetched, wether results are empty or not. Passes the data as parsed by parsedData if results are found, nothing otherwise. rejected: a network connexion failed to the server or a bug occured. Passes requirejs error.requireType param. if no result hits are returned the first time given the resistor values, it attempts to fetch new data with enlarged filters such as: 1- remove tolerance value from the filters - retry 2- if 1 not enough, enlarge resistor range values with RANGE_PERCENTAGE - retry 3- remove temperature value anyway This is performed in 3 steps as it is the limit of allowed octopart requests per second. We could decide to make more attemps by retrying after a setTimeout > 1s but filters enhancement from the 2 other requests should be enough.

    Parameters:

    • url String

      the raw octopart url

    Returns:

    Promise:

    the promise representing the requested url
